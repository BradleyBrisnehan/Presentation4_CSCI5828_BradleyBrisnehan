.NET Framework

One of the common uses of Microsoft Visual Studio is for the development of applications within the .NET framework. The .NET framework consists of two parts:

  • Common Language Runtime (CLR) is a just-in-time compiler used to translate compiled to machine code at execution time. All code written using the .NET framework is executed by the CLR independent of the language service written under. The CLR also offers automatic memory management (allocation of memory and garbage collection), thread control, exception handling, and common type declaration (among language services).
  • .NET Framework Class Library (FCL) defines a library of classes, interfaces, and types allowing access to Windows system functionality. The same library extends Windows functionality independent of the language service the code is written under.

During development, the .Net framework revision is defined. All workstations running the application must support a .NET version equal to or greater than the one specified during development.
